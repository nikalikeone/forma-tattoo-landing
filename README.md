# FORMA — landing page for a tattoo artist

Концепт одностраничного сайта для частного тату-мастера из Омска. Лендинг знакомит посетителя с визуальным стилем мастера, показывает работы, объясняет процесс и приводит к заявке на консультацию.

> Это самостоятельный демонстрационный проект для портфолио, а не опубликованная клиентская работа. Бренд, персонаж и изображения созданы для концепта. Перед коммерческим запуском их необходимо заменить реальными данными мастера.

## Задача

Посетители тату-мастера чаще всего приходят из поиска или социальных сетей и принимают решение по нескольким сигналам: уровню работ, доверию к мастеру, понятности процесса и простоте записи.

Сайт решает четыре задачи:

- показывает избранные работы в выразительной галерее;
- формирует доверие через рассказ о подходе, процессе и безопасности;
- отвечает на типовые вопросы до консультации;
- собирает исходные данные для обсуждения будущей татуировки.

## Основные возможности

- адаптивный интерфейс от 320 px;
- семантическая структура страницы;
- мобильное меню без обязательного JavaScript;
- асимметричная галерея работ;
- блоки о мастере, направлениях, процессе и безопасности;
- нативный раскрывающийся FAQ;
- форма с браузерной валидацией и загрузкой референса;
- отдельный шаблон политики обработки персональных данных;
- метаданные Open Graph и Twitter Card;
- условный canonical URL;
- динамические `robots.txt` и `sitemap.xml`;
- адаптивные WebP-изображения;
- локальные шрифты без зависимости от Google Fonts;
- поддержка `prefers-reduced-motion` и клавиатурной навигации.

## Технологии

- [Astro](https://astro.build/) — статическая сборка и компонентная структура;
- TypeScript — типизация конфигурации и клиентской логики;
- HTML5 — семантическая разметка;
- CSS — Grid, Flexbox, custom properties, `clamp()` и адаптивные состояния;
- Astro Image — генерация адаптивных форматов изображений;
- ESLint и Prettier — качество и единый стиль кода;
- Lighthouse — аудит производительности, SEO и доступности.

React и библиотека состояния здесь намеренно не используются: интерфейсу не требуется сложное клиентское состояние, а лишний JavaScript ухудшил бы загрузку.

## Структура проекта

```text
src/
├── assets/images/      # исходные изображения, обрабатываемые Astro
├── components/         # независимые секции лендинга
├── config/             # данные и SEO-конфигурация сайта
├── layouts/            # общий HTML-каркас и метаданные
├── pages/              # главная, политика, robots и sitemap
└── styles/             # глобальные токены и базовые стили
```

Каждая крупная секция вынесена в отдельный Astro-компонент. Это упрощает чтение, тестирование и будущую замену демонстрационного контента на клиентский.

## Локальный запуск

Требуется Node.js версии, указанной в `package.json`.

```bash
npm install
npm run dev
```

Production-сборка:

```bash
npm run build
npm run preview
```

## Проверки

```bash
npm run validate
```

Команда последовательно запускает:

1. проверку Astro и TypeScript;
2. ESLint;
3. проверку форматирования;
4. production-сборку.

## Настройка домена

Скопируйте `.env.example` в `.env` и укажите опубликованный адрес:

```env
PUBLIC_SITE_URL=https://example.com
```

После этого сборка добавит canonical URL, адрес сайта в Open Graph, а также ссылку на `sitemap.xml` в `robots.txt`.

## Что необходимо заменить перед реальным запуском

- название бренда;
- имя, опыт и специализацию мастера;
- фотографии реальных работ с разрешением на публикацию;
- подтверждённые сведения о безопасности;
- услуги, цены, сроки ответа и правила предоплаты;
- контакты и ссылки на социальные сети;
- данные оператора и условия обработки персональных данных;
- обработчик формы;
- домен и URL окружения;
- Open Graph-изображение;
- структурированные данные локального бизнеса.

Форма сейчас работает в демонстрационном режиме и не передаёт данные во внешний сервис.

## SEO и публикация

Техническая подготовка создаёт условия для индексирования, но не гарантирует конкретные позиции. После публикации необходимо:

1. подключить реальный домен и HTTPS;
2. проверить canonical URL, `robots.txt` и `sitemap.xml`;
3. добавить сайт в Яндекс Вебмастер и Google Search Console;
4. отправить карту сайта;
5. настроить региональность и карточки бизнеса;
6. добавить подтверждённые контакты и структурированные данные;
7. отслеживать индексацию и поисковые запросы;
8. развивать контент под реальные услуги мастера.

## Изображения и честность кейса

Все текущие изображения созданы специально для концепта с помощью генеративного инструмента и подписаны как демонстрационные. Они не являются татуировками или фотографиями реального заказчика.

## Лицензия

Проект предназначен для учебного портфолио. Перед повторным коммерческим использованием визуальной концепции, названия и материалов необходимо проверить права и возможные совпадения с существующими брендами.
